# Q5.1 - 实时数据管道设计

## 📋 题目背景

Olist 当前的数据是离线 CSV 导出。现在业务需求是——卖家希望**实时**看到自己的订单情况、买家评价、物流异常。

请设计一个**实时数据架构**，满足：

- 订单状态变更后 **30 秒内** 反映在卖家后台
- 评价文本进来后 **1 分钟内** 触发情感分析 + LLM 抽取
- 物流异常（如某订单延迟超过预计时间）**自动报警**到卖家
- 历史数据查询（卖家想看过去 6 个月的成交曲线）**毫秒级响应**
- 当前数据规模：**10 万订单/天**，预计 2 年内增长到 **100 万订单/天**

要求：800-1500 字 + 一张架构图
