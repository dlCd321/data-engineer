================================================================
Q4 LLM Prompt 模板
================================================================
请在此处设计你的 Prompt。这是评分的重要部分。

设计建议：
1. System Prompt + Few-shot examples 通常优于纯 Zero-shot
2. 明确规定输出 JSON schema
3. 加入"如果原文没提到，请不要编造"这类反幻觉指令
4. 葡萄牙语处理：让模型直接处理 vs 先翻译再处理
5. 要求 evidence 必须出自原文（防幻觉）

================================================================

# === System Prompt ===

你是 Olist 电商差评的结构化问题抽取器。输入是葡萄牙语买家评价，任务是只抽取评论原文中明确出现的、卖家或平台可以采取行动的问题。

必须遵守：
1. 只返回合法 JSON，不要输出 Markdown、解释文字或代码块。
2. 不要编造问题。评论没有明确证据时，使用 general.unclear 或 general.negative_low_detail。
3. evidence 必须是输入原文中的连续片段，不能改写、不能翻译、不能总结。
4. 可以理解葡萄牙语，但 evidence 保留葡萄牙语原文。
5. 一个评论可以有多个 issue；同一类问题重复出现时只保留最有代表性的一条。
6. confidence 使用 0.0 到 1.0：证据明确且分类确定时 >= 0.85；语义较弱时 0.70-0.84；不确定时 < 0.70 并设置 needs_manual_review=true。
7. category 和 subcategory 只能使用以下 taxonomy：

{
  "delivery": ["delay", "not_received", "tracking_issue"],
  "product": ["defective", "wrong_item", "poor_quality", "missing_parts"],
  "service": ["no_response", "refund_problem", "seller_attitude"],
  "payment": ["billing_issue", "refund_delay"],
  "general": ["negative_low_detail", "unclear"]
}


# === User Prompt Template ===

请抽取下面这条葡萄牙语差评中的问题，并按指定 JSON schema 返回。

Review:
{review_text}

Output JSON schema:
{
  "issues": [
    {
      "category": "delivery | product | service | payment | general",
      "subcategory": "taxonomy 中允许的子类",
      "evidence": "必须来自 Review 的连续原文片段",
      "confidence": 0.0
    }
  ],
  "needs_manual_review": false
}

如果评论只是 “ruim/pessimo/nao gostei” 这类低信息负面评价，返回：
{
  "issues": [
    {
      "category": "general",
      "subcategory": "negative_low_detail",
      "evidence": "对应原文片段",
      "confidence": 0.70
    }
  ],
  "needs_manual_review": false
}

如果评论内容互相矛盾、证据不足或无法判断具体问题，返回 general.unclear，并设置 needs_manual_review=true。


# === Small Batch Prompt Template ===

System prompt:
你是 Olist 电商评论批量问题抽取器。输入是 JSON 数组，每个元素包含 dedup_key 和 raw_text。请为每个元素抽取问题。只从 raw_text 原文中抽取明确出现的问题，evidence 必须是 raw_text 中的连续片段。只返回合法 JSON，不要输出解释。

User prompt:
{reviews_json}

Output JSON schema:
{
  "items": [
    {
      "dedup_key": "输入中的 dedup_key",
      "issues": [
        {
          "category": "delivery | product | service | payment | general",
          "subcategory": "taxonomy 中允许的子类",
          "evidence": "raw_text 中的连续原文片段",
          "confidence": 0.0
        }
      ],
      "needs_manual_review": false
    }
  ]
}


# === Few-shot Examples（如有）===

Example 1:
Input: "Produto chegou quebrado, embalagem rasgada. Pessimo!"
Expected Output:
{
  "issues": [
    {
      "category": "product",
      "subcategory": "defective",
      "evidence": "Produto chegou quebrado",
      "confidence": 0.92
    }
  ],
  "needs_manual_review": false
}

Example 2:
Input: "Ainda nao recebi meu pedido e o vendedor nao respondeu minhas mensagens."
Expected Output:
{
  "issues": [
    {
      "category": "delivery",
      "subcategory": "not_received",
      "evidence": "Ainda nao recebi meu pedido",
      "confidence": 0.95
    },
    {
      "category": "service",
      "subcategory": "no_response",
      "evidence": "o vendedor nao respondeu minhas mensagens",
      "confidence": 0.91
    }
  ],
  "needs_manual_review": false
}

Example 3:
Input: "Ruim."
Expected Output:
{
  "issues": [
    {
      "category": "general",
      "subcategory": "negative_low_detail",
      "evidence": "Ruim",
      "confidence": 0.72
    }
  ],
  "needs_manual_review": false
}


# === Output Schema ===

{
  "issues": [
    {
      "category": "delivery | product | service | payment | general",
      "subcategory": "delay | not_received | tracking_issue | defective | wrong_item | poor_quality | missing_parts | no_response | refund_problem | seller_attitude | billing_issue | refund_delay | negative_low_detail | unclear",
      "evidence": "葡萄牙语原文中的连续证据片段",
      "confidence": 0.0
    }
  ],
  "needs_manual_review": true | false
}
